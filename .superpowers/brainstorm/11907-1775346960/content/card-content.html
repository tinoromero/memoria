How should question & answer content be structured?

Each "side" of a flashcard can carry different types of content. Pick the model that fits best.

A

Text only, OR Image only

Each side is either a text field or an image — not both. The user picks one mode per side.

Card front
text or image
Card back
text or image

Pros

  • Simplest model
  • Clean UI — one or the other

Cons

  • Can't combine text + image on same side
  • Less expressive
B

Text (required) + Image (optional) ⭐ Recommended

Each side always has a text field. An image can optionally accompany it. At least the text must be filled.

Card front
question text ✱ required
+ image (optional)
Card back
answer text ✱ required
+ image (optional)

Pros

  • Text always available for search, lists, previews
  • Images enrich without replacing text
  • Simple validation — just require the text

Cons

  • Can't have image-only cards (e.g. "identify this diagram")
C

Text (optional) + Image (optional) — at least one required

Maximum flexibility. Each side can be text only, image only, or both. Validation just requires at least one to be filled.

Card front
text (optional)
image (optional)
✱ at least one required
Card back
text (optional)
image (optional)
✱ at least one required

Pros

  • Fully flexible — image-only cards supported
  • Covers all use cases

Cons

  • Image-only cards have no text for search or list previews
  • Slightly more complex validation

Images will be stored in Supabase Storage (private bucket, per-user folder). The DB stores the image path, not a public URL.
