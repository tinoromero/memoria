Data Model

Supabase (PostgreSQL) tables and relationships. Does this cover everything you need?

auth.users — managed entirely by Supabase Auth. Contains id, email, and Google OAuth metadata. All tables reference auth.users(id) via user_id.
TABLE topics
id uuid PK
user_id uuid FK → auth.users
name text, not null
created_at timestamptz, default now()
TABLE questions
id uuid PK
user_id uuid FK → auth.users
topic_id uuid FK → topics (nullable)
question text, not null
answer text, not null
streak int, default 0
is_memorized bool, default false
created_at timestamptz, default now()
TABLE sessions
id uuid PK
user_id uuid FK → auth.users
topic_ids uuid[], nullable (null = all topics)
question_count int, not null
include_memorized bool, default false
completed_at timestamptz, nullable
created_at timestamptz, default now()
TABLE session_questions
id uuid PK
session_id uuid FK → sessions
question_id uuid FK → questions
result enum: correct | wrong | pending
position int — display order in session
Scoring logic (client-side): When a question is marked correct → streak + 1. If streak ≥ 10 → is_memorized = true. When marked wrong → streak = 0. Updates happen at the end of each session via a single batch write to Supabase.
Weighted selection logic (server-side): Questions are selected using weighted random sampling — weight = 1 / (streak + 1). So streak 0 → weight 1.0, streak 5 → weight 0.17, streak 9 → weight 0.1. Memorized questions excluded unless include_memorized = true.
